## 1. 研究对象（What）

### 当前板块

站点覆盖 **8 个顶级板块**（均为"AI × 具体产业"的交叉）：

| 板块 | 核心问题 |
|---|---|
| 人工智能 | 基础模型、Agent、技术 / 基础设施 / 生态本身 |
| 机器人 | 硬件本体 + 产业链（减速器 / 丝杠 / 电机） |
| 具身智能 | 机器人大脑（VLA / VLM / World Model） |
| AI 医疗 | 药物发现 / 影像 / 临床 Copilot / 合成生物 |
| 自动驾驶 | L3/L4 / Robotaxi / 端到端自驾 |
| 脑机接口 | 侵入式 / 非侵入 / 医疗 / 消费 / 神经伦理 |
| AI 金融 | 量化 / 投研 / 风控 / 保险 / 合规 |
| AI 互联网 | 搜索 / 浏览器 / Coding / 应用层 |

外加一个 **方法论板块**（框架库 10 篇 + 基石 + 信息源工具）。

### "够格成为独立板块"的判断标准

不是所有"AI × X"都够独立。判断三条硬标准：

1. **旗舰报告中有独立章节**：Stanford HAI / Benaich State of AI 等至少一份把它作为独立章节处理（例：Stanford HAI 2025 有独立的 Science and Medicine 章节 → AI 医疗够格）
2. **独立玩家群 ≥ 8 家**：有 8 家以上公司专注做这一方向、能写出完整公司档案（Tesla Optimus 不算具身智能玩家——它是机器人整机）
3. **与现有板块不能合并**：如果能放进已有板块的一个子维度，就不独立
   - ❌ AI 硬件设备（Humane / Rabbit）：规模小，放 AI 互联网就够
   - ❌ AI 教育：Khan Academy / Duolingo 规模不够撑独立板块
   - ✅ AI 医疗：规模大、玩家独立、与任何现有板块差异明显
   - ✅ 自动驾驶：万亿美元市场，玩家独立，不适合并在机器人或 AI 互联网下

### 候选观察（暂未独立，按需升格）

- **AI 国防与国家安全**（Anduril / Palantir AIP / Shield AI）：敏感，暂不独立
- **AI 内容生产**（Suno / Runway / ElevenLabs / HeyGen）：当前在 AI 互联网下，未来若突破可升格
- **AI 教育**（Khanmigo / Duolingo / Speak）：当前放 AI 互联网垂直

## 2. 研究方法（How）

### 每篇文章的统一结构

所有深度文（2500-3500 字）按以下 8 段式组织：

1. **摘要 / TL;DR**（单句结论，20 字内）
2. **3-5 条关键要点**（每条一句话）
3. **事实基线**（5 个硬数字 + 来源，强调时点 / 单位 / 地域）
4. **框架应用**（显式命名一个框架，例：用 Wardley Maps 分析）
5. **分层分析**（价值链 / 玩家 / 商业模式 / 地缘 / 监管多视角）
6. **签名图表**（Mermaid / 表格，设计为可截图转发）
7. **前瞻与自评承诺**（2-3 条可证伪前瞻 + 日期）
8. **延伸阅读**（人工精挑 3 条）+ **信息源**（分层）

### 前瞻写法的纪律

前瞻**必须可证伪**：

- ✅ "2026 上半年六小虎中至少一家退出基础模型"
- ❌ "AI 会继续发展"（废话）

每条前瞻附：
- 观察指标（如何判断成真）
- 打分日期（何时回来复盘）

## 3. 分类方法（Why this taxonomy）

### 核心原则：**按"内容维度"而非"研究方法"**

**不用**："行业研究 / 公司研究 / 产品研究"这三分法
**用**：每个板块内部按 3-10 个**分析视角**（市场 / 技术 / 基础设施 / 应用 / 地缘 / 政策 / 安全 / 人才 ...）扁平化组织

### 为什么

调研了 30+ 份旗舰报告（Stanford HAI / Benaich State of AI / Menlo Enterprise AI / McKinsey / BCG / Gartner / a16z / SemiAnalysis / Stratechery / Sequoia）：

- **0/13 AI 报告**使用"公司研究"作为独立章节
- **0/10 非-AI 行业报告**使用"产品研究"作为独立章节
- 专业报告**全部按内容维度组织**，公司 / 产品作为**证据嵌入各章**

"行业/公司/产品"三分法源自**中国散户金融自媒体 / 券商研报 genre**，不是学术或国际行业研究的标准做法。

### 本站的实施

每个板块在 `docs/0X_<板块>/` 下：

- **01_ 到 10_**：内容维度子目录（按板块特色）
- **11_公司研究**：实体索引（作为辅助检索方式保留）
- **12_产品研究**：实体索引

维度数量视板块内容密度而定（人工智能 10 个，AI 金融 3 个，具身智能 2 个）。

## 5. 文件与 URL 命名

### 文件名

- 单文件名 **≤ 5-8 字**
- 短中文 / 拼音 / 英文均可，**不堆砌多个子主题**
- ❌ `机器人核心零部件供应链_谐波减速器_丝杠_电机.md`
- ✅ `核心零部件.md`

### Nav 显示名与文件名解耦

```yaml
- 核心零部件：减速器 · 丝杠 · 电机: path/to/核心零部件.md
```

- Key（nav 显示）：清爽完整的中文描述，可带标点
- Value（文件路径）：短文件名

### 目录排序

用 `0X_` 前缀控制：
- `01_-10_`：内容维度
- `11_公司研究`
- `12_产品研究`
- `99_方法论`（跨板块）

## 6. 质量门禁

发布任何改动前必须满足：

- [ ] `mkdocs build --strict` 零 warning（无破链、无孤儿页）
- [ ] 信息源分层标注（最强 / 强 / 中）
- [ ] 每条事实性陈述带来源（财报 / 长访谈 / 官方博客 / 一手论文）
- [ ] 数字带时点 / 单位 / 地域
- [ ] 观点段用"我的判断"前缀，与事实段明确区分

### 事实三个不要

AI 辅助写作时绝对**不从 AI 直接生成**：

- 具体数字（营收 / 估值 / 融资）
- 具体人名（"X 是 Y 公司 CTO"）
- 具体时间（"2024-03 发布 Z"）

这些必须从原始出处（财报 PDF / 官方新闻稿 / LinkedIn）复制粘贴。

## 7. 更新节奏

| 频率 | 事项 |
|---|---|
| **重大事件 24h 内** | 发快评（格式：500-800 字原子卡片；当前未开设专门板块）|
| **每周 1-2 篇** | 常规内容补充 |
| **每季度** | 更新灯塔公司页（估值 / 最新动态） |
| **每半年** | 更新行业总览（AI 格局 / 人形机器人格局等） |
| **每年 1 月** | 发布当年度预测（12 条可证伪）|
| **每年 12 月** | 对当年度预测自评打分 |

### 年度锚仪式

仿 Benaich State of AI 做"预测 + 次年自评"。这个仪式是 **inbound 链接和读者忠诚度最高效的单一产品动作**。
